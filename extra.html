Console-Based Monitoring

In linux monitoring active process can be done by using ps command. ps command create a snapshot of the currenlty running active process and show it to the user’s screen. ps allow users to view details such as process IDs (PIDs), CPU and memory usage, parent process IDs (PPIDs), and more

here are some parameter used in ps to show more in-dept review of a active process.

  • Basic Usage: Running ps without any options displays a snapshot of processes owned by the current user in the current terminal session.
    ps
  • Display All Processes: To display information about all processes on the system, including those owned by other users and system processes, use the e option.
    ps -e
  • Display Detailed Process Information: The -f option provides a more detailed output, including additional information such as the user who owns the process, the process ID (PID), the parent process ID (PPID), CPU and memory usage, and more.
    ps -ef

Display Specific Process: To display information about a specific process, specify the PID using the -p option.

ps -p 1180

  • top: top is a command-line tool that provides real-time information about system processes, CPU usage, memory usage, and more. It displays a dynamic list of running processes and their resource utilization. Pressing keys like k, r, and q allows you to kill processes, renice processes, and quit top, respectively.
  • htop: htop is an enhanced version of top with a more user-friendly interface and additional features. It provides colorized output, mouse support, and the ability to scroll horizontally and vertically. htop also allows you to interactively manage processes using function keys.

Using the kill Command to Control Processes.

Kill command used to kill the active process which is running in the system. In Linux every process have a unique process id called PID in-short. with the help of PID we will able to kill the particular process in the Linux system.

setting up telnet

Setting up a Telnet server on Linux involves installing the Telnet server software, configuring it to start automatically, and optionally configuring access controls and security settings. Here's a basic guide to setting up a Telnet server on Linux:

  1. Install Telnet Server:
    • First, you need to install the Telnet server software. On Debian-based systems like Ubuntu, you can install it using the following command:
      sudo apt-get update
sudo apt-get install telnetd
  1. Start Telnet Service:
  • Once installed, start the Telnet service. On systemd-based systems (most modern Linux distributions), you can use the following command:
    sudo systemctl start telnet.socket
  • To ensure the Telnet service starts automatically on boot, enable it with:
    sudo systemctl enable telnet.socket
  1. telnet configuration
  • By default, Telnet allows access to all users on the system. You may want to restrict access to specific users or groups. This can be achieved by modifying the Telnet server configuration file.
  • On Debian-based systems, the configuration file is located at /etc/default/telnetd. You can edit this file to specify allowed users or groups.
  1. Telnet Connection:

Once the Telnet server is set up, establish the telnet connection by using telnet as a command followed with the ip-address. it will prompt you for credential such as username and password to get login as a telnet

telnet <server_ip_or_hostname>

Using Network Configuration tools

  1. netctl : Netclt provide the ip address to the host in dynamic or static way. netctl uses profiles to manage network connections and different modes of operation to start profiles automatically or manually on demand. The netctl profile files are stored in /etc/netctl/ and example configuration files are available in /etc/netctl/examples/

    To configure the dhcp, make a profile in location such as /etc/netctl/my_dhcp_profile

    Interface=enp1s0
Connection=ethernet
IP=dhcp

    To configure Static ip address, make a profile in location such as /etc/netctl/my_static_profile

    Interface=enp1s0
Connection=ethernet
IP=static
Address=('10.1.10.2/24')
Gateway='10.1.10.1'
DNS=('10.1.10.1')

    Once you have created your profile, attempt to establish a connection, where profile is only the profile name, not the full path:

     netctl start profile_name

    A profile can be enabled to start at boot by using:

    # netctl enable profile

    NetworkManager is a dynamic network configuration daemon that provides a high-level interface for managing network connections. It is commonly used in desktop environments and

    Configuring Samba with system-config-samba

    Samba is an open-source implementation of the Server Message Block (SMB) protocol suite. SMB protocol used in the Windows to used share file and printers on local network. Using Samba you can make your Linux machine look just like a Windows computer to other Windows computers on your network.

    System-config-samba is a program to configure samba in easy way, to install in on debian linux type following command

    sudo apt install samba system-config-samba

    Now launch the application, this application run as a super user to configure samba, type your superuser password and the application will be pop-up on your screen.

    Setting up the application is stright-forward click on the + sign to create a samba share basic setting. In the basic settings you need to provide the path to the folder that you want to share via
    Samba. You also need to give it a share name, and an optional description. If you plan on
    setting up a number of shares then you may want to consider filling out the description
    to help you distinguish between them all.

    Next up you need to select one or both of the check boxes to allow users to view (visible)
    and or write (writeable) to the folder.

    now restart the service to make change in the samba

    sudo systemctl restart smbd


    That's it! Your Samba shares should now be configured according to your settings in
    system-config-samba. You can test access to your shares from other machines on your network to ensure everything is working as expected.


    Configuring Samba with SWAT

    Configuring Samba with SWAT (Samba Web Administration Tool) provides a web-based interface for managing Samba configuration. Here's how you can set it up:

    First, ensure that Samba and SWAT are installed on your Linux system. You can typically install them using your package manager. For example, on Debian/Ubuntu systems, you can use:

    sudo apt-get install samba swat


    By default, SWAT is not enabled to listen on any network interfaces due to security reasons. To enable SWAT, you need to modify the Samba configuration file (
    /etc/samba/smb.conf). Open the file in a text editor.

    [global]
    ...
    web port = 901
    interfaces = lo eth0  # Adjust this according to your network interfaces

    After making changes to the configuration file, restart the Samba service to apply the changes

    sudo systemctl restart smbd


    Open a web browser and navigate to
    http://localhost:901 or http://your_server_ip:901. You should see the SWAT login page. Log in using your system credentials (username and password). Ensure that you have sufficient privileges to access SWAT.Once logged in, you can use the SWAT interface to configure various aspects of your Samba server, including shares, users, printers, and global settings. The interface is self-explanatory and offers help tooltips for each configuration option.

    After making changes, ensure to save and apply them through SWAT.If required, restart the Samba service again to apply the changes made via SWAT

    Remember to keep SWAT secured by only allowing access from trusted networks and ensuring that it's protected with proper authentication mechanisms. Additionally, always backup your Samba configuration before making significant changes.
